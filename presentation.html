Introducing

Research → Plan → Implement

A three-phase workflow that turns AI-assisted development from a guessing game into a repeatable process.

Research Plan Implement

The problem with "just ask the AI"

  • 🎲 The agent jumps straight to code — skipping the thinking that makes the code correct.
  • 🔄 Feedback loops are expensive: wrong decisions surface late, when changing them is hardest.
  • 📭 Nobody knows why a decision was made — not the agent, not the developer.
  • 💥 Each new session starts from scratch. Context collapses. Work gets repeated.
The core insight

Each phase needs a different cognitive mode

Research Divergent — explore, question, gather facts
→
Plan Convergent — evaluate, decide, commit
→
Implement Procedural — execute, verify, deliver

Mixing these modes in a single prompt — phase collapse — is the root cause of most AI-assisted development failures.

Phase 1

Research

Before writing a single line of code, the agent reads the codebase and maps all unknowns. The operator then decides which to resolve — and whether to answer them directly or let the Plan phase address them.

Artifact: research.md
Criterion: FAR — Factual · Actionable · Relevant

Exit gate: every open question has a cited, factual answer
Anti-pattern: collecting opinions instead of facts
Phase 2

Plan

Research findings drive a task list where every task is independently testable. No task may be added that isn't grounded in the research.

Artifact: plan.md
Criterion: FACTS — Feasible · Atomic · Clear · Testable · Scoped

Exit gate: each task has a pass/fail acceptance criterion
Anti-pattern: a plan that is documentation theater
Phase 3

Implement

The agent executes only what the plan prescribes, marks each task pass or fail, and produces a signed-off record of what was built.

Artifact: source code + SIGNOFF
Criterion: each task from plan.md resolves to pass or fail

Exit gate: all tasks resolved; SIGNOFF written
Anti-pattern: scope creep, undocumented deviations
The cardinal rule

One phase per session. Always.

Phase Artifact Validation
Research research.md FAR — every answer is a citable fact
Plan plan.md FACTS — every task is atomic and testable
Implement code + SIGNOFF Each task resolves to pass or fail

The operator — not the agent — decides when to move to the next phase.

Meet the toolkit

RPI Kit

A single installable skill that enforces the RPI workflow in any AI agent — Copilot, Claude, Gemini, Cursor, Codex.

SKILL.md

Lightweight router — loads only the instructions for the active phase. No token waste on inactive stages.

stages/

Three self-contained files, one per phase. Each is the complete, authoritative guide for that phase.

resources/

Templates for research.md and plan.md. Validation guide with FAR and FACTS criteria.

rpi-new.sh

Scaffolds a new project folder with today's date and a slug. Copies the research template to get started.

Getting started

Two commands, any agent

# 1. Install the skill once
bash install.sh

# 2. Tell your agent which phase to run
@rpi-workflow Research "My Feature"

The agent scaffolds the project folder automatically, then reads the stage file, produces the artifact, and stops. You review, then call the next phase when ready.
