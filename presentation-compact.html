Research → Plan → Implement

A three-phase workflow that turns AI-assisted development from a guessing game into a repeatable process.

Research Plan Implement

Why "just ask the AI" breaks down

  • 🎲 The agent jumps straight to code — skipping the thinking that makes the code correct.
  • 🔄 Wrong decisions surface late, when they're most expensive to reverse.
  • 💥 Each new session starts from scratch. Context collapses. Work gets repeated.

The fix: separate the thinking from the doing.

Three phases. Three cognitive modes.

Phase 1 Research

Explore the codebase and map all unknowns. The operator decides which to resolve — and whether to answer them directly or defer to the Plan phase.

FAR criteria
Phase 2 Plan

Research findings drive a task list where every task is atomic and independently testable.

FACTS criteria
Phase 3 Implement

Execute only what the plan prescribes. Each task resolves to pass or fail. Produce a SIGNOFF.

Pass / Fail
The toolkit

RPI Kit — install once, use everywhere

SKILL.md

Lightweight router — loads only the instructions for the active phase. No token waste.

stages/

Three self-contained files, one per phase. The complete guide for that phase and nothing else.

resources/

Templates for research.md and plan.md. Validation guide with FAR and FACTS criteria.

rpi-new.sh

Scaffolds a dated project folder automatically when the agent starts a Research phase. No manual step needed.

Works with any agent: Copilot, Claude, Gemini, Cursor, Codex.
